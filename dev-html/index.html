Sistema de Controle de Versões Git e Repositório GitHub

Introdução

O Git é um sistema de controle de versões distribuído amplamente utilizado no desenvolvimento de software. Ele permite que equipes de desenvolvedores colaborem em um projeto, controlando as alterações nos arquivos ao longo do tempo. O GitHub é uma plataforma de hospedagem de código que utiliza o Git para permitir o compartilhamento e a colaboração em projetos.

Git

O Git oferece uma variedade de comandos para gerenciar o controle de versões em um projeto. A seguir, estão listados alguns dos comandos mais utilizados:

Comandos Git

  • git init: Inicializa um repositório Git em um diretório.
  • git clone: Clona um repositório Git existente para o seu ambiente local.
  • git branch: Cria, lista ou exclui branches no repositório.
  • git checkout: Alterna entre branches ou restaura arquivos para uma versão específica.
  • git status: Exibe o status dos arquivos no repositório.
  • git diff: Mostra as diferenças entre commits, branches ou arquivos.
  • git add: Adiciona arquivos para serem commitados.
  • git commit: Registra as mudanças no repositório.
  • git push: Envia as alterações locais para um repositório remoto.
  • git pull: Atualiza o repositório local com as alterações do repositório remoto.
  • git revert: Desfaz um commit específico criando um novo commit.
  • git merge: Combina alterações de uma branch para outra.
  • git stash: Armazena mudanças temporariamente para poder alternar de branch.
  • git rm: Remove arquivos do repositório.
  • git config: Configura opções do Git.
  • git reset: Desfaz alterações em um commit específico.
  • git remote: Gerencia repositórios remotos.
  • git fetch: Busca as alterações do repositório remoto para o repositório local.
  • git show: Mostra informações sobre um commit específico.
  • git help: Exibe a documentação de ajuda do Git.
  • git rebase: Reaplica commits em uma branch a partir de outra.
  • git blame: Mostra quem fez as alterações em um arquivo.
  • git tag: Marca pontos específicos no histórico de commits.

GitHub

O GitHub é uma plataforma online que permite hospedar repositórios Git remotamente e oferece recursos adicionais para a colaboração em projetos. Para utilizar o GitHub, siga os seguintes passos:

  1. Crie uma conta no GitHub em https://github.com/.
  2. Crie um novo repositório no GitHub, definindo um nome e uma descrição.
  3. No seu ambiente local, utilize o comando git clone para clonar o repositório vazio para o seu diretório de trabalho.
  4. Adicione, modifique e organize os arquivos do seu projeto localmente utilizando os comandos do Git.
  5. Utilize os comandos git add e git commit para preparar e registrar as alterações no seu repositório local.
  6. Use o comando git push para enviar as alterações para o repositório remoto no GitHub.
  7. Para sincronizar as alterações feitas por outras pessoas, utilize o comando git pull.
